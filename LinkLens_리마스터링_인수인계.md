# LinkLens 리마스터링 인수인계 문서
> 작성 기준: 원본 프로젝트 전체 분석 + 리마스터링 방향 통합  
> 목적: 새 AI와 리마스터링 작업 시 UI·기능·AI 고급화 방향을 빠르게 공유

---

## 1. 서비스 개요

**LinkLens**는 URL을 저장하면 AI가 자동으로 제목·요약·키워드·카테고리를 추출해주는 개인 지식 관리(PKM) SaaS다.  
핵심 가치: "링크를 붙여넣기만 하면 나머지는 AI가 알아서 정리"

| 항목 | 내용 |
|------|------|
| 배포 | Cloudflare Pages |
| 백엔드 | Cloudflare Workers (AI 처리) + Supabase (Auth + DB) |
| AI | OpenAI GPT-4o-mini |
| 콘텐츠 파싱 | Jina Reader API (본문), 직접 HTML fetch (제목) |

## 3. 화면별 상세 UI

### 3-1. 인증 화면

- 전체 화면 오버레이 (`display: flex`로 중앙 정렬)
- 상단: 로고 + 서비스명
- 탭 바: **로그인** / **회원가입** (탭 전환 시 폼 교체, 탭 하이라이트)
- 로그인 폼: 이메일 input + 비밀번호 input + 로그인 버튼
- 회원가입 폼: 이메일 input + 비밀번호 input + 비밀번호 확인 input + 회원가입 버튼
- 에러 메시지 영역: 폼 하단 빨간 텍스트
- 성공 메시지 영역: 초록 텍스트
- 처리 중: 버튼 비활성화 + "처리 중..." 텍스트
- 전체 배경: 블러 처리된 그라데이션 또는 단색

**에러 케이스별 메시지:**

| 상황 | 메시지 |
|------|--------|
| 잘못된 비밀번호 | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 이미 가입된 이메일 | "이미 가입된 이메일입니다. 로그인 해주세요." (+ 로그인 탭 자동 전환) |
| 비밀번호 불일치 | "비밀번호가 일치하지 않습니다" |
| 회원가입 완료 | "가입이 완료되었습니다. 로그인해주세요." (초록) |

### 3-2. 사이드바

**고정 너비 232px, 스크롤 독립, 항상 표시**

```
[로고]                    [테마 토글 🌙/☀️]

─ 라이브러리 ─────────────────────
  전체 기사          [23]
  미읽음             [5]
  나중에 읽기         [3]
  즐겨찾기            [8]
  휴지통             [2]

─ 카테고리 ──────────────────────
  테크/IT            [10]
  경제/금융           [6]
  AI/미래            [4]
  과학               [2]
  정치/사회           [1]

─ 컬렉션 ────────────────────────
  🔵 오프쇼어 윈드     [5]  ✏️ 🗑️
  🟢 투자 아이디어     [3]  ✏️ 🗑️
  [+ 컬렉션 추가]

─ 하단 ──────────────────────────
  Free 플랜
  ████████░░ 23/50개
  [Pro로 업그레이드]
```

**동작 규칙:**
- 각 항목 클릭 → 해당 필터 적용, 활성 항목 하이라이트
- 숫자 뱃지: 해당 필터 기준 기사 수 실시간 반영
- 컬렉션: 색상 점(●) + 이름 + 수량, 우측에 수정(✏️)·삭제(🗑️) 버튼 (hover 시 표시)
- 컬렉션 추가: 이름 입력 + 색상 선택 인라인 폼
- 컬렉션 수정: 동일한 인라인 폼 재사용
- 테마 토글: 다크↔라이트, localStorage에 저장
- Free 플랜 바: 저장된 기사 수 / 최대 수 표시

### 3-3. Topbar

```
[현재 뷰 제목 "전체 기사"]    [🔍 검색...  ⌘K]    [BL 배일성▼]  [+ 링크 추가]
```

- 현재 선택된 사이드바 항목명이 제목으로 표시
- 검색창: 포커스 시 확장, ⌘K 단축키로 바로 포커스
- 유저 버튼: 이메일 첫 글자 아바타 + 이름(이메일 @ 앞부분)
- 유저 드롭다운 (클릭 시): 이메일 전체 + [로그아웃] 버튼
- "링크 추가" 버튼: 강조색 배경, 클릭 시 모달 오픈

### 3-4. 콘텐츠 영역

**통계 카드 4개 (상단)**

```
┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐
│ 전체 기사   │ │  미읽음    │ │ AI 요약완료 │ │  즐겨찾기  │
│    23      │ │     5      │ │    20      │ │     8      │
└────────────┘ └────────────┘ └────────────┘ └────────────┘
```

**툴바**
```
정렬: [최신순 ▼]    [미읽음만 □]    [■■] [≡]
```
- 정렬 옵션: 최신순 / 오래된순 / 별점순
- 미읽음만 필터: 토글 체크박스
- 뷰 토글: 카드뷰(■■) / 리스트뷰(≡)

**카드뷰 - 기사 카드 구성:**
```
┌─────────────────────────────────────┐
│ [썸네일/파비콘]  카테고리 뱃지        │
│ 제목 (2줄 클리핑)                    │
│ 출처 · 날짜                          │
│ 요약 (3줄 클리핑)                    │
│ [키워드1] [키워드2] [키워드3]         │
│ ────────────────────────────────── │
│ [상태 뱃지]  ★★★☆☆   [⭐] [🗑️]    │
└─────────────────────────────────────┘
```
- 상태 뱃지: 미읽음(파란)/나중에읽기(노란)/읽음(회색)
- 즐겨찾기(⭐): 카드에서 직접 토글, 채워진 별 = 즐겨찾기 on
- 삭제(🗑️): 카드에서 직접 삭제 → 휴지통 이동
- 카드 클릭: 상세 패널 오픈 + 자동 읽음 전환
- AI 분석 중 카드: 요약 영역에 "AI 분석 중..." 스켈레톤 또는 텍스트

**리스트뷰 - 기사 행 구성:**
```
[파비콘] 제목 (1줄)  |  출처  |  날짜  |  카테고리  |  상태  |  ★  |  [⭐][🗑️]
```

**빈 상태:**
- 일러스트 + "아직 저장된 링크가 없습니다" + "링크 추가하기" 버튼

**휴지통 뷰 상단 바:**
```
⚠️ 휴지통에 있는 기사는 브라우저에만 저장됩니다.  [휴지통 비우기]
```
- 기사 카드에 [복원] [영구삭제] 버튼만 표시 (즐겨찾기·삭제 대신)

### 3-5. 링크 추가 모달

```
┌──────────────────────────────────────────────┐
│  🤖 AI가 자동으로 제목·요약·키워드를 추출합니다    │
├──────────────────────────────────────────────┤
│  URL                                         │
│  [https://...                    📋 붙여넣기] │
│                                              │
│  제목 (자동 감지)                              │
│  [                                         ] │
│                                              │
│  카테고리              읽기 상태               │
│  [테크/IT ▼]          [미읽음 ▼]              │
│                                              │
│  태그                                        │
│  [#태그입력 Enter]  [기존태그 ×]               │
│                                              │
│  메모                                        │
│  [                                         ] │
├──────────────────────────────────────────────┤
│                    [취소]  [AI 분석 후 저장 🤖] │
└──────────────────────────────────────────────┘
```

**동작 규칙:**
- URL 입력 600ms 후 `/preview` 호출 → 제목 자동 입력 (로딩 스피너 표시)
- 제목 직접 수정 시 이후 자동완성 중단 (`manualEdit` 플래그)
- 저장 중: 버튼 스피너 + 상단 프로그레스 바
- 저장 성공: 모달 닫힘 + 카드 즉시 표시 + "저장되었습니다. AI 분석 중..." 토스트
- AI 분석 완료: 카드의 요약/키워드 업데이트 + "AI 분석 완료" 토스트
- AI 분석 실패: 저장은 유지 + "AI 분석에 실패했지만 링크는 저장되었습니다" 토스트

### 3-6. 기사 상세 패널

**우측에서 슬라이드인, 반투명 오버레이 클릭 시 닫힘**

```
┌─────────────────────────────────────┐
│ ← 뒤로                           ⭐ │
├─────────────────────────────────────┤
│ [썸네일 이미지 or 파비콘 아이콘]       │
│                                     │
│ 제목 (전체 표시, 줄바꿈 허용)          │
│ 출처 · 날짜                          │
│ [카테고리 뱃지]                       │
│                                     │
│ ─ AI 요약 ─────────────────────     │
│ 요약 텍스트 (전체)                    │
│                                     │
│ ─ 키워드 ──────────────────────     │
│ [키워드1] [키워드2] [키워드3]...       │
│                                     │
│ ─ 태그 ────────────────────────     │
│ [태그 ×] [태그 ×]  [+ 태그 추가]      │
│                                     │
│ ─ 컬렉션 ──────────────────────     │
│ [컬렉션 선택 ▼]                      │
│                                     │
│ ─ 별점 ────────────────────────     │
│ ★★★☆☆ (클릭 가능)                   │
│                                     │
│ ─ 메모 ────────────────────────     │
│ [메모 textarea, 자동 저장]            │
│                                     │
├─────────────────────────────────────┤
│ 상태: [미읽음][나중에읽기][읽음]        │
│                                     │
│ [🗑️ 삭제]              [원문 보기 →] │
└─────────────────────────────────────┘
```

**동작 규칙:**
- 패널 열릴 때 자동으로 `unread → read` 전환 (status DB 업데이트)
- 즐겨찾기·별점·태그·컬렉션·메모·상태 변경 시 즉시 DB 저장 (debounce 없음, 메모만 onChange)
- 태그: Enter 또는 쉼표로 추가, `#` 자동 제거, × 클릭으로 삭제
- 컬렉션: 다중 배정 가능 (기사 하나가 여러 컬렉션에 속할 수 있음)
- 삭제: confirm 다이얼로그 없이 즉시 휴지통 이동
- 원문 보기: 새 탭으로 URL 오픈

## 4. 전체 기능 목록

### 4-1. 인증 (P0)
| 기능 | 설명 |
|------|------|
| 이메일 로그인 | 이메일 + 비밀번호 |
| 이메일 회원가입 | 이메일 + 비밀번호 + 확인 |
| 세션 자동 유지 | 브라우저 재방문 시 자동 로그인 |
| 로그아웃 | 드롭다운 → 로그아웃 버튼 |
| 중복 가입 감지 | identities 배열 확인 → 에러 메시지 + 로그인 탭 전환 |

### 4-2. 기사 저장 (P0)
| 기능 | 설명 |
|------|------|
| URL 저장 | URL 입력 → 즉시 DB 저장 (title 임시값 포함) |
| 제목 자동완성 | URL 입력 600ms 후 /preview 호출 → og:title 자동 입력 |
| AI 분석 | 저장 후 백그라운드에서 /analyze 호출 → 제목·요약·키워드·카테고리 업데이트 |
| 카테고리 선택 | 저장 시 수동 선택 가능, AI가 자동 분류도 가능 |
| 상태 선택 | 저장 시 미읽음/나중에읽기/읽음 선택 |
| 태그 입력 | 저장 시 태그 미리 입력 가능 |
| 메모 입력 | 저장 시 메모 미리 입력 가능 |
| 중복 저장 방지 | savingArticle 플래그로 버튼 중복 클릭 차단 |

### 4-3. 기사 조회 및 탐색 (P1)

## 5. AI 파이프라인 상세

### 현재 구조

```
[사용자 URL 입력]
      │
      ▼ (600ms 디바운스)
POST /preview
  └─ 직접 HTML fetch → og:title / twitter:title / <title> 추출
  └─ 응답: { title }
      │
[모달에 제목 자동완성]
      │
[저장 버튼 클릭]
      │
      ▼ (즉시)
DB INSERT (title=임시값, status=pending 등)
      │
      ▼ (백그라운드, 비동기)
POST /analyze
  ├─ Promise.allSettled([직접HTML fetch, Jina fetch])
  │   ├─ HTML: og:title 추출 (제목 확정)
  │   └─ Jina: 본문 마크다운 추출
  └─ OpenAI GPT-4o-mini 호출
      system: 제목·요약·키워드·카테고리 JSON 반환 지시
      user:   "원문 제목: {fetchedTitle}\n\n{본문텍스트}"
      │
      ▼
DB UPDATE (title, summary, keywords[], category)
      │
[카드 UI 업데이트 + "AI 분석 완료" 토스트]
```

### Worker GPT 시스템 프롬프트 (원본 정책)

```
다음 JSON만 반환 (마크다운 코드블록 금지):
{
  "title":    "string — 원문 제목 그대로 (영어면 한국어 번역)",
  "summary":  "string — 3~4문장 핵심 요약 (한국어)",
  "keywords": ["string", ...] — 핵심 키워드 5~8개,
  "category": "테크/IT" | "경제/금융" | "AI/미래" | "과학" | "정치/사회" | "기타"
}

title 규칙:
- 원문 제목 제공 시 그대로 사용 (영어면 번역)
- 없을 경우 본문 핵심 요약 제목 생성
- 클릭베이트 없이 내용 반영
```

## 6. AI 고급화 방안

> 새 프로젝트 구조를 건드리지 않고 Worker와 프론트 UI만 확장하는 방식으로 제안

### 6-1. 분석 품질 향상

**현재 문제:** GPT에게 본문 전체를 넘기면 긴 기사는 잘리거나 핵심을 놓침

**개선안:**
- 본문을 청크 단위로 나눠 중요 문장 추출 후 GPT에 전달 (Jina의 응답 길이 제한 우회)
- 요약 길이를 사용자가 설정 가능하게 (1문장 / 3문장 / 상세)
- 카테고리 커스텀 허용 (사용자가 자신만의 카테고리 정의)

### 6-2. 읽기 경험 향상

**요약 고급화 옵션 (패널 UI 추가):**

```
─ AI 요약 ───────────────────────── [⚙️ 재분석]
| 3문장 요약 (기본)                             |
| [1문장] [3문장] [상세] ← 사용자 선택           |
```

- 재분석 버튼: 해당 기사만 Worker 재호출
- 요약 스타일 선택: 짧게(1문장) / 보통(3문장) / 상세(불릿 5개)

### 6-3. 스마트 분류

**태그 자동 추천:**
- AI 분석 결과에 `suggestedTags` 필드 추가
- 패널에서 추천 태그를 클릭 → 즉시 추가

```
─ 태그 ─────────────────────────────
| [에너지] [해상풍력] ×               |
| AI 추천: [재생에너지] [ESG] [탄소중립]  |  ← 클릭으로 추가
```

**관련 기사 추천:**
- 같은 키워드/카테고리에서 읽지 않은 기사 3개 패널 하단에 표시

### 6-4. 검색 고급화

**현재:** 클라이언트 사이드 문자열 매칭  
**개선안:** Supabase `fts` (full-text search) 활용으로 서버 사이드 검색
- `tsvector` 컬럼 추가 (title + summary + keywords + tags)
- 검색 쿼리를 DB에서 처리 → 대용량에서도 빠름

### 6-5. AI 요약 캐싱 / 재사용

- 같은 URL을 다른 유저가 저장 시 AI 재분석 생략 가능 (공용 캐시 테이블)
- `url_cache (url, title, summary, keywords, category, cached_at)` 테이블 추가
- 동일 URL 분석 시 캐시 히트 → Worker가 바로 반환 → 비용·속도 개선

### 6-6. 배치 분석 (미래 기능)

- 기존에 AI 분석 실패했던 기사를 "재시도" 버튼으로 일괄 분석
- 설정 페이지에서 "분석 안 된 기사 모두 분석" 버튼

### 6-7. 읽기 타임 예측

- 본문 단어 수 기반으로 "약 3분 읽기" 표시
- Worker에서 `readingTime` 필드 추가 반환

## 7. 데이터 구조

### articles 테이블
```sql
id          bigint       PK, auto-increment
user_id     uuid         FK → auth.users, RLS 기준
url         text         원본 URL
title       text         기사 제목 (AI 확정 전: 임시값)
source      text         출처 도메인 (예: naver.com)
category    text         카테고리 (AI 분류)
summary     text         AI 요약
keywords    text[]       키워드 배열
tags        text[]       사용자 태그 배열
status      text         'unread' | 'later' | 'read'
starred     boolean      즐겨찾기
rating      int          1~5 (null = 미평가)
memo        text         사용자 메모
collections bigint[]     컬렉션 ID 배열 (다중 배정)
date        text         기사 날짜 (파싱된 것, 없으면 저장일)
created_at  timestamptz  저장 시각
```

### collections 테이블
```sql
id          bigint       PK
user_id     uuid         FK → auth.users
name        text         컬렉션 이름
color       text         색상 코드 (예: #3B82F6)
created_at  timestamptz
```

### RLS 정책 (두 테이블 공통)
```sql
-- SELECT
CREATE POLICY "user reads own" ON articles
  FOR SELECT USING (auth.uid() = user_id);

-- INSERT
CREATE POLICY "user inserts own" ON articles
  FOR INSERT WITH CHECK (auth.uid() = user_id);

-- UPDATE
CREATE POLICY "user updates own" ON articles
  FOR UPDATE USING (auth.uid() = user_id);

-- DELETE
CREATE POLICY "user deletes own" ON articles
  FOR DELETE USING (auth.uid() = user_id);
```

## 8. 상태 전이 규칙 (정책)

### 읽기 상태
```
unread → read    : 상세 패널 열 때 자동
unread → later   : 패널 또는 저장 시 수동
later  → read    : 패널 열 때 자동
any    → any     : 패널 상태 버튼으로 수동 전환 가능
```

### 삭제/복원
```
기사 삭제 → DB에서 즉시 삭제 + localStorage trash에 객체 저장
휴지통 복원 → DB에 새 INSERT (새 id) + trash에서 제거
영구삭제 → trash에서만 제거 (DB에는 이미 없음)
"휴지통 비우기" → trash 배열 전체 초기화
```

### AI 분석 실패 시
```
저장은 유지 (DB에 title만 있는 상태)
토스트: "AI 분석에 실패했지만 링크는 저장되었습니다"
재분석: 현재 UI 없음 → AI 고급화 방안 6-6에서 추가 권장
```

### 컬렉션 삭제 시
```
collections 테이블에서 해당 row 삭제
articles.collections 배열에서 해당 ID 제거 (모든 관련 기사 UPDATE)
사이드바 뱃지 즉시 갱신
```

| 항목 | 이유 |
|------|------|

| authState 세션 관리 플래그 | Supabase v2 이벤트 중복 발화 방지용 |

## 11. 새 AI와 협업 시 전달할 핵심 컨텍스트

```
이 프로젝트는 URL 북마크 + AI 요약 SaaS입니다.

핵심 흐름:
1. 사용자가 URL 입력
2. /preview 호출 → 제목 자동완성
3. 저장 버튼 → DB INSERT 즉시 + /analyze 백그라운드 비동기 호출
4. AI 분석 완료 → DB UPDATE + UI 카드 갱신

DB: Supabase PostgreSQL
- articles 테이블 (keywords, collections가 배열 타입)
- RLS: user_id = auth.uid() 모든 작업에 적용

AI 처리: Cloudflare Worker
- /preview: og:title만 빠르게 반환
- /analyze: HTML + Jina 병렬 fetch → GPT-4o-mini 분석 → title+summary+keywords+category 반환

UI 구조: 사이드바(필터) + 콘텐츠(카드/리스트) + 우측 상세패널
- 사이드바 항목: 라이브러리(전체/미읽음/나중에읽기/즐겨찾기/휴지통) + 카테고리 + 컬렉션
- 카드에서 즐겨찾기·삭제 가능
- 패널에서 상태·태그·컬렉션·별점·메모 편집 → 즉시 DB 저장

휴지통: localStorage 기반 (DB 아님) → 기기 변경 시 소실

현재 미해결 버그 3개:
1. 새로고침 후 기사 0개 (Supabase 세션 이벤트 중복 발화)
2. 로그아웃 직후 재로그인 멈춤 (지연된 SIGNED_OUT 이벤트)
3. URL 저장 무한 로딩 (Worker 인증/CORS 문제 추정)
```
